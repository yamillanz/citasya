# CitasYa - Design Context

## Users
Business owners/managers and employees in small service businesses (beauty salons, dental clinics, massage parlors). Primary context: clients accessing the booking system via mobile devices. Secondary context: managers and employees using the backoffice on both mobile and desktop for daily operations, appointment management, and financial tracking.

## Brand Personality
**Professional, efficient, trustworthy** - The interface should feel like a reliable business tool that respects users' time. No playful or whimsical elements. Calm and efficient interactions that convey competence and reliability.

## Aesthetic Direction
**Minimal, elegant, organic** - Clean lines with subtle organic touches (rounded corners, soft shadows). Light mode exclusively. Current color palette: sage green (#9DC183) as primary, warm neutrals (cream, linen, sand), and coral accents (#E07A5F) for functional colors. Typography: Fraunces for display headings (elegant serif), DM Sans for body text (clean sans-serif). Avoid screaming colors, gradients for text, or overly decorative elements.

## Design Principles
1. **Mobile-first clarity**: Design for mobile screens first, ensure touch targets are adequate, and information hierarchy is clear on small screens.
2. **Consistent pattern library**: Use the established design tokens (CSS custom properties) and component patterns from STYLES.MD. Never deviate from the token system.
3. **Functional minimalism**: Every element must serve a purpose. Avoid decorative elements that don't enhance usability or brand recognition.
4. **Organic warmth**: Use the existing warm neutrals and sage green palette to create a calm, professional atmosphere. Subtle shadows and rounded corners provide organic feel without clutter.
5. **Performance-conscious**: Animations should be subtle and purposeful (fadeIn, scaleIn). Avoid heavy effects that impact mobile performance.

## Additional Notes
- Existing STYLES.MD is the definitive reference for all design decisions
- PrimeNG components must be used with the established override patterns
- No Tailwind classes in templates (despite Tailwind being configured)
- All new components must follow the SCSS structure outlined in STYLES.MD
- Responsive breakpoints are defined in STYLES.MD - follow those patterns exactly
